# PTEPrep

PTE Academic exam practice platform. Monorepo with React web app, Cloudflare Worker backend, and a planned React Native mobile app.

## Language

**Question**:
Core domain entity representing a PTE Academic exam item, one of 22 types
(read*aloud, repeat_sentence, retell_lecture, describe_image, respond_situation,
write_essay, summarize_written, summarize_discussion, reading_fib_dragdrop,
reading_fib_dropdown, reorder_paragraphs, reading_mcs, reading_mcm,
highlight_incorrect, highlight_summary, listening_mcs, listening_mcm,
listening_fib, select_missing_word, write_from_dictation, summarize_spoken_text,
answer_short). Stored in the `questions` table with a `question_type` enum.
\_Avoid*: Item, task, exercise

**Blueprint**:
JSONB column on the questions table — a Gemini-generated AI recipe (intent,
tone, complexity, CEFR level) used to produce Clones. Not Apeuni data.
_Avoid_: Prompt, template, AI config

**Clone**:
An AI-generated question (`is_ai_generated=true`, `is_public=true`,
`tenant_id=NULL`). Visible to all public users. Points to its parent Real
question via `cloned_from`.
_Avoid_: Generated question, AI question

**Real question**:
An Apeuni-sourced question (`is_ai_generated=false`). Has a non-null
`source_ref`. Visible only to the owning Tenant or never made public.
_Avoid_: Original question, Apeuni question, source material

**Tenant**:
A B2B training center (e.g., an English coaching institute). Owns Real
questions via `tenant_id` foreign key on `questions`.
_Avoid_: Organization, school, account, customer

**source_ref**:
Private Apeuni UID stored as JSONB on the `questions` table. Used only for
internal traceability. Must never be exposed via any API endpoint —
intentionally omitted from the `public_questions` view and all query results.
_Avoid_: External ID, foreign key, upstream ref

**Signed URL**:
A time-limited R2 URL (TTL ≤ 3600s) generated by the Worker for audio/image
media. The frontend never holds a permanent media URL.
_Avoid_: Pre-signed URL, token URL, temporary link

## Dev Data Warning

A small subset of Real questions (`is_ai_generated=false`,
`source_ref IS NOT NULL`) have been temporarily set to `is_public=true`
in the development database for UI development purposes. This is a
dev-only state.

PRODUCTION RULE (never violate): A question with `source_ref IS NOT NULL`
must NEVER have `is_public=true` in production. `source_ref` indicates
Apeuni-sourced content which cannot be publicly served.

Before any production deployment, run:

```sql
UPDATE questions SET is_public = false WHERE source_ref IS NOT NULL AND is_public = true;
```

The Worker's `GET /v1/questions` endpoint must additionally filter with
`AND (is_ai_generated = true OR source_ref IS NULL)` as defence-in-depth —
never rely solely on `is_public=true` being set correctly.

This warning exists because the schema legitimately allows `is_public=true`
on Real questions (for future edge cases) so the DB constraint alone
cannot catch this mistake.
